# 管理界面布局改进

## 问题描述

原管理界面存在以下问题：
1. 内容区域过小，没有充分利用屏幕空间
2. 缺少导航菜单，无法在不同页面间切换

## 解决方案

### 1. 创建统一布局组件

创建了 `src/app/admin/pool/layout.tsx` 文件，提供：

- **侧边栏导航**：包含仪表盘、账号管理、日志查看三个菜单项
- **响应式设计**：
  - 桌面端：固定侧边栏（宽度 256px）
  - 移动端：可折叠侧边栏，通过汉堡菜单打开
- **退出登录**：底部提供退出登录按钮
- **当前页面高亮**：自动高亮当前访问的页面

### 2. 更新页面布局

修改了以下页面，移除了外层容器，使用布局提供的空间：

- `src/app/admin/pool/page.tsx` - 仪表盘
- `src/app/admin/pool/accounts/page.tsx` - 账号管理
- `src/app/admin/pool/logs/page.tsx` - 日志查看

### 3. 修复构建错误

- 修复了 `global-pool.ts` 中 DBManager 实例化问题（使用 `getInstance` 方法）
- 修复了 `login/page.tsx` 中的 HTML 转义字符问题

## 特性

1. **全屏布局**：内容区域占据除侧边栏外的所有空间
2. **平滑过渡**：侧边栏展开/收起有动画效果
3. **一致性**：所有管理页面使用统一的布局和导航
4. **可访问性**：支持键盘导航和屏幕阅读器

## 技术细节

- 使用 Tailwind CSS 实现响应式布局
- 使用 Next.js App Router 的 layout 功能
- 使用 `usePathname` 和 `useRouter` 实现导航和路由
- 使用 `localStorage` 存储认证信息
